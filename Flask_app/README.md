# LINA - Web Interface

LINA is a web-based dashboard and control interface for a robotics project.

## Recent Updates
- Fixed layout issues across the Cockpit and Debug pages where flex containers (like the system logs and console output) would expand infinitely and break the layout boundary grid rows. This was resolved by properly implementing `min-height: 0` on flex items and grid containers to permit the content to scroll.
- **Major UI Redesign:** Replaced the default camera video feed interface with a purely sensor-driven dashboard featuring a rotating algorithmic radar UI (simulating the rotating ultrasonic head), a frontal proximity HUD array, and visual 3-state IR indicators. All sensor simulations natively operate on the frontend via Canvas APIs and Javascript timed loops for immediate UI verification.
- **Drive & Calibrate UI Update:** Replaced the complex Nipple.js 360-degree joystick with a discrete, 5-button D-pad interface (Up, Down, Left, Right, Stop) aligned with the custom cyberpunk styling. Additionally, replaced the manual Head Pan slider with fixed-value preset buttons (0°, 45°, 90°, 135°, 180°).
- **Backend Rewrite (`app.py`):** Completely restructured with a structured `robot_state` dict, a `broadcast_telemetry()` helper, safety-guarded D-pad control handler, and full bidirectional UDP integration for ESP32 hardware communication (sending `MOVE`, `SERVO`, `SET_MODE`, and `SET_SPEED` JSON commands).
- **Auto Scan Feature:** Added an Auto Scan toggle button to the head pan controls. When active, the robot's head will continuously sweep back and forth (0° to 180°), updating the sensor dashboard dynamically.
- **Military Radar UI:** The rotating radar dashboard now features a classic military green color scheme with glowing red square markers for detected objects.
- **Single-Sensor Refactor (2026-04-28):** LINA now has ONE ultrasonic sensor (head-mounted, servo-controlled). `front_dist` is always aliased to `head_dist` — no independent front sensor. Dashboard relabelled from "FRONT PROXIMITY" to "HEAD SONAR". Obstacle forward-block now only triggers when `head_angle` is between 70°–110° AND `head_dist < 20 cm`.
- **Robot Chat & Side Panel:** Added a new side panel to the left of the 3D robot model. This section hosts a real-time **Robot Chat** interface powered by LINA's internal brain (Groq LLM + Smart Search) and a more accessible **Speed Control** slider.
- **UI Layout Optimization:** Relocated the Speed Control from the bottom console to the side panel to resolve visibility issues and provide a more intuitive "Drive & System" control layout. The dashboard now uses a flexible horizontal layout that adapts better to different screen sizes.
- **Chat Integration:** Integrated `chatbot.py` with `app.py` via SocketIO, enabling users to ask LINA questions about the world, time, weather, or robot status directly from the dashboard.
- Fall detection added: if ESP32 sends `"fall": true`, Flask auto-sends STOP command and the cockpit shows a red flash overlay alert for 4 seconds. The `fall` boolean is now included in all `telemetry_update` SocketIO broadcasts.
- **Bug Fix:** Fixed `getaddrinfo failed` socket error in `app.py` by removing the incorrect leading whitespace from the `ESP32_IP` string.
- **ESP32 Firmware Bug Fixes:** Fixed IP misconfiguration in `esp32_lina.ino` by removing the static IP constraint (which placed the ESP32 on an unreachable `172.30.158.x` subnet instead of the host PC's `192.168.1.x` Wi-Fi subnet) and updated `FLASK_IP` to the PC's actual local IP (`192.168.1.4`). Also fixed a potential UDP packet read buffer overflow and corrected an issue where the robot would remain permanently blocked from moving forward if the head turned away from an obstacle.
- **UDP Broadcast Fallback:** Updated `app.py` to default `ESP32_IP` to `255.255.255.255` (broadcast mode). This ensures that drive and pan commands still reach the ESP32 on the local network even if inbound telemetry is blocked by Windows Firewall, bypassing the IP mismatch trap.
- **3D Robot Avatar Integration:** Replaced the 2D green military radar scanning UI in the cockpit dashboard with a real-time, interactive 3D procedural robot model using Three.js. The 3D robot's head rotates synchronously with the physical robot's `head_angle` telemetry data, and the avatar's tyres rotate in real-time syncing with drive commands (forward, backward, left, right).
- **Auto-Resume Movement:** Refactored obstacle avoidance logic across `app.py` and `esp32_lina.ino`. The ESP32 now tracks the "intended direction" so that if the robot is blocked while moving forward, it physically stops, but automatically resumes forward movement the moment the obstacle is cleared. The UI's 3D wheels also pause and resume automatically to match the physical state.
- **Dynamic 3D Obstacle Warning:** The 3D robot model's "eyes" now actively monitor the telemetry feed. Whenever the head sonar detects an obstacle within 9cm (in any direction during scanning), the eyes instantly flash red and the proximity dashboard illuminates with the exact distance.
- **Real-Time Speed Control:** Added a premium speed control slider to the dashboard. This allows the user to adjust the robot's movement speed (0-100%) dynamically, which is forwarded via the ESP32 to the motor controller.
- **Mobile-First Cockpit Redesign (2026-05-08):** Completely overhauled the mobile interface for both portrait and landscape views. The new 'Widget Grid' architecture ensures that 100% of desktop features (Calibration, Logs, Missions, and Navigation) are accessible on a single screen without any scrolling. Optimized the 3D HUD, Chat, and Drive clusters for ultra-high density displays, ensuring a premium, low-latency experience on smartphones.
- **Micro-Animation & Glassmorphism:** Applied glowing borders, scanline effects, and glassmorphism across all mobile widgets to create a unified, high-tech aesthetic.
- **Offline Voice Commands (Vosk Integration):** Added complete offline Speech-to-Text capability using Vosk and PyAudio. A new `voice_handler.py` module continuously captures microphone input locally on the backend. When the dashboard is switched to **VOICE** mode, spoken commands (like "Move forward" or chat queries) are transcribed and routed through the `chatbot.py` brain, automatically triggering physical robot movement or a conversational reply completely offline.
